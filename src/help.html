How do I open the extension?

Using the keyboard shortcut

Press Ctrl + Shift + S (or Cmd + Shift + S on Mac) to open the SenseUI panel.

Pinning the extension to your toolbar

  1. Open Chrome's extension menu (located in the toolbar palette next to the address bar)
  2. You can also access Chrome's extensions by going to chrome extensions page(opens in new tab)
  3. Find SenseUI in the list of your extensions
  4. Select "details"
  5. Toggle the "pin to toolbar" option

Can I change the keyboard shortcut?

Yes! You can customize the keyboard shortcut by opening the browser's extension manager. Click the "Open shortcuts page" button below to access it directly.

Important notes:

  • If you choose a keyboard shortcut that conflicts with a browser native shortcut, your custom shortcut will override the browser's default.
  • Resetting SenseUI settings will not reset this keyboard shortcut. To change it, you must return to Chrome's shortcuts page.

What commands are available?

SenseUI provides quick commands for common tasks. Type / in the input field to access them.

Available commands

  • /describe: Generates a visual description of the page
  • /issues: Identifies design issues ranging from critical accessibility problems to mild visual inconsistencies
  • /refresh: Re-captures page data (HTML, CSS, and screenshot)
  • /clear: Clears the current chat session

Can I ask custom questions?

Yes! You can ask anything about your webpage by typing a custom prompt directly into the input field at the bottom of the SenseUI panel. Simply focus the field, type your question or request, and press Enter to submit.

How can I navigate the chat?

User and System responses are structured with semantic headings that you can navigate using your screen reader's heading navigation shortcuts.

What actions can I perform on messages?

Each AI response includes action buttons (functionality will be implemented in a later sprint):

  • Copy to Clipboard
  • Mark as Favorite. This flags important messages to selectively download when you export your chat history.

How does SenseUI analyze my page?

SenseUI captures three key pieces of information from your webpage:

  • HTML structure: The semantic markup and content of your page
  • CSS styles: The visual styling rules applied to your elements
  • Screenshot: A visual capture of how your page actually renders

The AI analyzes these together to provide accurate, context-aware feedback about your page's design, accessibility, and usability. It cross-references what the code says with what's actually visible in the screenshot to ensure its analysis matches reality.

How does the screenshot modes work?

Choose which part of the page SenseUI captures for analysis:

Viewport only (Default)

Captures only what's currently visible in your browser window. This mode provides faster feedback, uses fewer tokens, and offers more detailed analysis. Best for analyzing specific sections or working iteratively on parts of your page.

Full page

Captures the entire page from top to bottom. This mode is slower and uses more tokens, but provides a broader overview of your entire page layout. Not ideal for infinite scroll pages or sites with heavy dynamic content.

Note: By default, SenseUI only analyzes what's visible in your viewport. If you're using the /describe command and want to see the whole page, switch to Full page mode in Settings.

How do I use Projects?

Projects allow you to provide context about your website so SenseUI can give framework-specific and design-aligned feedback. When a project is active, SenseUI considers your specified technologies, aesthetic goals, and website purpose in its analysis and recommendations.

Creating a Project

  1. Navigate to the Projects page from the navigation menu
  2. Fill out the "Create New Project" form with the following required information:
    • Project Name: A memorable name for your project
    • Framework & Technologies: Specify the main frameworks and technologies you're using (e.g., "React, Tailwind CSS"). You can select from suggestions or type your own.
    • Desired Aesthetic: Describe the visual style you're aiming for (e.g., "Minimalist and professional", "Modern and vibrant")
    • Website Type & Purpose: What is this website for? (e.g., "E-commerce platform", "Portfolio website", "Blog")
  3. Press the "Create Project" button
  4. You'll hear a confirmation that your project was created successfully

Activating a Project

To receive project-specific feedback:

  1. Go to the Chat page (main page)
  2. Locate the "Active Project" dropdown selector
  3. Select your desired project from the list
  4. You'll hear a confirmation that the project context is now active

Once activated, all AI responses will consider your project's frameworks, aesthetic, and purpose when providing feedback.

Managing Your Projects

On the Projects page, you can:

  • View all projects: Your projects are listed under "Your Projects" section
  • Edit a project: Navigate to the Edit button for the project you want to modify, press it, update the information in the form, and press "Save Changes"
  • Delete a project: Navigate to the Delete button for the project you want to remove, press it, and confirm the deletion in the dialog. This action cannot be undone.
  • Deactivate project context: In the Chat page, select "No project selected" from the Active Project dropdown to return to generic feedback mode

When to Use Projects

Projects are most useful when:

  • You're working on a specific website with defined design goals
  • You want recommendations that align with your chosen framework or technology stack
  • You need feedback consistent with a particular aesthetic or design direction
  • You're building for a specific purpose (e.g., e-commerce) and want relevant best practices

For quick, one-off analyses of random websites, you can skip creating a project and use SenseUI without an active project context.

Can I choose which AI model SenseUI uses?

Yes. On the Settings page, under "Enter your AI Key", you can:

  • Select your provider (OpenAI or Google Gemini)
  • Enter your API key for that provider
  • Choose a model from a suggestion list or type a supported model name

SenseUI is currently tested with a small set of focused, grounded models:

  • OpenAI: auto (gpt-4o-mini), gpt-4o, gpt-4.1-mini
  • Gemini: auto (gemini-3-flash-preview), gemini-3-pro-preview

Newer GPT-5 family models are not supported yet, because they do not allow lowering the temperature below 1.0. SenseUI uses a lower temperature to keep answers closely tied to the actual page content instead of more random or speculative responses.

How does SenseUI use my data?

SenseUI prioritizes your privacy. Here's exactly what happens:

Installation and code transparency

SenseUI is distributed through the Chrome Web Store and is open source. The complete source code is publicly available on GitHub (opens in new tab), allowing anyone to audit exactly what the extension does.

Data stored locally in your browser

The following data is stored only on your device using Chrome's local storage:

  • API keys: Stored encrypted and never shared with anyone except the AI provider you selected (OpenAI or Google Gemini)
  • Settings: Your preferences like screenshot mode, selected AI provider, and model choice
  • Chat history: Your conversation with SenseUI, stored only in your browser
  • Projects: Your project details including frameworks, aesthetics, and purpose

This data stays on your device and is browser-specific. It's not synced across devices, not uploaded to any servers, and is not accessible to the extension developers or anyone else.

Data sent to AI services

When you ask SenseUI to analyze a page, the following is sent directly from your browser to your chosen AI provider (OpenAI or Google Gemini):

  • The HTML structure of the page
  • The CSS styles of the page
  • A screenshot of the page (viewport or full page, depending on your settings)
  • Your question or prompt
  • Your active project context (if you have one selected)

Important privacy guarantees

  • No SenseUI servers: We don't operate any servers and don't collect, store, or have access to any of your data. All communication goes directly from your browser to the AI provider you choose.
  • AI provider policies: Data sent to OpenAI or Google Gemini is subject to their respective privacy policies. Review them before use:
    • OpenAI Privacy Policy (opens in new tab)
    • Google Privacy Policy (opens in new tab)
  • Your responsibility: Do not share confidential, proprietary, or sensitive information through SenseUI. If your code contains API keys, secrets, or private data, remove them before analysis.
  • Uninstalling: If you uninstall SenseUI from the Chrome Web Store, all locally stored data (settings, chat history, projects) is automatically removed from your browser.

For more details about data protection in the research context, see the Data Protection documentation (opens in new tab).
